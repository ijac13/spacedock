The refit skill has ~18 hardcoded "CL" references that should be "the captain" or "the user." The generated first-officer at .claude/agents/first-officer.md also has hardcoded "CL" (fixable by refit once the skill is fixed). Entity files using `source: CL` are data, not a code issue.

A second developer trying to use Spacedock shouldn't be addressed as "CL."

## Problem Statement

`skills/refit/SKILL.md` contains 16 hardcoded references to "CL" (a specific user's name) throughout the skill prompt. These should use the Spacedock domain term "the captain" — the person operating the pipeline. A new user running `/spacedock refit` would encounter instructions referencing "CL" instead of themselves.

## Scope

**Only file affected:** `skills/refit/SKILL.md` (16 occurrences)

No other plugin source files (skills/commission, templates, scripts) contain hardcoded "CL" references. The commission skill and templates already use `__CAPTAIN__` placeholder or generic terms correctly.

**Out of scope:**
- `source: CL` in entity frontmatter — this is data (who filed the task), not a code issue
- `.claude/agents/first-officer.md` — gets regenerated by refit; will be fixed once the skill is fixed
- Templates using `__CAPTAIN__` — already correct

## Proposed Approach

Replace all 16 occurrences of "CL" in `skills/refit/SKILL.md` with "the captain" (Spacedock's domain term for the pipeline operator). All 16 references are in the same context: interacting with the person running the pipeline (showing diffs, asking for confirmation, presenting results). No occurrence requires "the user" — "the captain" is the right term throughout because the refit skill is always operating in a pipeline context.

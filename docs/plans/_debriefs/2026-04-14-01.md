Items discovered after the debrief was committed that affect the historical record:

- **Teams is NOT enabled on CI by default.** Empirical check across 5 fo-log.jsonl files from PR #92's failed cycle 4 run: every FO `init` event lists zero team-related tools (no `TeamCreate`, `SendMessage`, `TeamDelete`). Claude Code's `curl https://claude.ai/install.sh` install on CI does not enable agent teams. The FO correctly enters bare mode for every dispatch on CI; `claude-team build` is invoked with `bare_mode: true` and the helper correctly suppresses the `### Completion Signal` block. This **invalidates the "FO paraphrases the helper output" bullet under Issues — Spacedock above** — that diagnosis was wrong. The actual failure mode for `test_dispatch_completion_signal` was that the test asserted team-mode behavior unconditionally while CI runs bare-mode-only. Cycle 5 fixed the test (mode-aware assertion) and trimmed the cycle-4 prose ("Verbatim character-for-character", anti-patterns list, post-dispatch verification) that was warning about a failure mode that wasn't happening.

Returns `None` when token file is absent so CI behavior is unchanged. This addresses the "run_first_officer doesn't isolate HOME" bullet under Issues — Spacedock; the fix is in PR #92 (not yet merged), no separate task needed.

Trimming prose can be a real fix.

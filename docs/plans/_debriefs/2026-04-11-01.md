Shipped the fix as task 131/PR #75. The **upstream Claude Code stripping behavior** remains and may warrant a separate upstream issue.
2. **`tests/test_agent_content.py` stale assertion (129 regression).** 129's merge removed wording that 125's test still asserted. No CI caught it. Shipped the fix inside 117/PR #78. Root cause (no CI for static tests) tracked in task 133.
3. **076 stale reference paths.** `tests/test_feedback_keepalive.py:307` and `scripts/test-harness.md:163,166,169,172` pointed at pre-076 `references/` paths. Shipped the fix inside 117/PR #78. Part of the "no CI for static/E2E tests" pattern.
4. **`test_scaffolding_guardrail.py` false positive.** Violation pattern matches any `Bash(ls -la ...)` or other read-only bash commands containing a scaffolding path. Tracked in task 134.
5. **`test_rejection_flow.py` haiku 600s timeout.** Multi-cycle rejection pipelines don't finish within budget on haiku. Tracked in task 134.
6. **Agent `name=` parameter footgun.** Claude Code's Agent tool schema marks `name` as optional, but named-team dispatch requires it — without it, subagents don't join the team and go idle as task-scoped transcripts. Half of this session ran before spotting it. Tracked in task 120 (filed previously).

## Issues — Spacedock

1. **Dead ensigns cannot be cleanly shut down.** `SendMessage(shutdown_request)` is cooperative — it requires the target to process the request. Dead agents silently drop it; zombies persist in team config until session end. Documented in task 121 Gap 2. Workaround: fresh-dispatch with `-cycleN` suffix and track dead ensigns in session memory. Upstream gap.

2. **`isSidechain:true` is not a bug marker.** Every subagent jsonl in this session has `isSidechain:true` regardless of actual team membership. FO nearly raised a false alarm reading this field. Reliable marker is team config membership. Documented in 121's runtime reference.

3. **Agent tool `model` enum does not accept `[1m]` variant.** Tested: `model: "claude-opus-4-6[1m]"` is rejected at input validation. Parent session's `[1m]` does not propagate. Workaround per anthropics/claude-code#40929: `teammateDefaultModel: null` in `~/.claude.json` + `/model opus[1m]` in the lead. Applied to CL's environment; unverified on v2.1.96.

4. **`gh pr edit --body` fails with Projects Classic deprecation.** REST API `PATCH` workaround used twice. Probably an upstream `gh` issue, not Spacedock.

5. **FO broke pr-merge mod flow on 116.** Operational error: direct-merged after a rebase conflict on accumulated stage reports, bypassing the mod. Captain flagged, FO reverted and went through proper flow. Root cause was the entity-body accumulation problem, now fixed by task 125.

## Problem

The scoring system has two issues:

1. **Over-engineered default.** The 5-dimension rubric (Edge/Fitness/Parsimony/Testability/Novelty, each 1-5, sum out of 25) is domain-specific and imposes unnecessary complexity on new pipelines. Most pipelines just need a simple signal: is this worth doing or not?

2. **Inconsistent format across the codebase.** The "/25" suffix appears in the spec and commission skill but actual entity files use plain integers. Agents reading different docs get conflicting guidance.

## Proposed Approach

1. **Default scoring is 0-1 (floating point).** New pipelines get `score:` as an optional field with values from 0.0 to 1.0. This is a continuous range — 0.0 means not worth doing, 1.0 means highest priority, and values in between express relative priority (e.g., 0.3 = low priority, 0.7 = high priority). No rubric, no dimensions — just a simple prioritization signal. The scoring rubric section in the commission skill becomes optional, not generated by default.

2. **Explicit upgrade path.** The commission skill and spec should document that scoring can be upgraded as a project matures. A pipeline that needs more rigorous prioritization can adopt a multi-dimension rubric (like this pipeline's 5x5 system) by updating its README schema. This is a conscious choice, not the default.

3. **Fix format inconsistencies.** Standardize on plain number in all docs: `score: 0.8` not `score: 1/25` or `(score: 22/25)`. Update `v0/spec.md` and `skills/commission/SKILL.md` to match.

### Files to modify

- `v0/spec.md` — change score format from "N/25" to plain integer, document 0-1 default and upgrade path
- `skills/commission/SKILL.md` — default to 0-1 scoring, make rubric optional, fix format references
- `docs/plans/README.md` — this pipeline already uses the 5-dimension rubric; no change needed (it's an upgraded scoring system)

## Open Questions (Resolved)

- **Q: Should scoring be mandatory?** A: No. The schema already marks score as optional.
- **Q: What about existing entities with 1-5 dimension scores?** A: They keep their scores. The 5-dimension rubric is this pipeline's chosen upgrade. The change only affects the default for NEW commissions.

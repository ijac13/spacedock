Two process guardrails for the first-officer template:

1. **Issue-first guardrail:** Do not make direct fixes to templates, skills, or scaffolding files. File a GitHub issue (with captain approval) or create a pipeline task first, then implement referencing it.

2. **Issue filing guardrail:** Do NOT run `gh issue create` without explicit captain approval. Present the draft issue title and body to the captain and wait for confirmation before filing.

See GitHub issue #4 for context.

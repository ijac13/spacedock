# Spacedock Development — Agent Guidelines

## Self-hosted pipeline boundaries

Spacedock uses its own pipeline (`docs/plans/`) to manage development. This creates a distinction between project source code and pipeline artifacts:

**Ensigns may edit (project source code):**
- `skills/` — commission skill and other plugin skills
- `templates/` — first-officer and other generated templates
- `scripts/` — test harnesses, utilities
- `v0/`, `plugin.json` — plugin source

**Ensigns must NOT edit (pipeline artifacts):**
- `docs/plans/README.md` — pipeline schema, owned by refit
- `docs/plans/*.md` frontmatter — entity state, owned by first officer
- `.claude/agents/` — generated by commission/refit, not direct editing

**Why:** In a normal project using Spacedock, this distinction is obvious — ensigns edit project code, not pipeline config. In Spacedock's own repo, the project code *is* pipeline infrastructure, so the line needs to be explicit.
